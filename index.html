What This Project Is Really About

The first high scores were not the interesting part. The real work was making the testing setup stricter and more realistic, then seeing which ideas still helped after that.

What ended up helping most

  • looking at heartbeat shape at more than one time scale
  • looking at nearby beats instead of judging one beat alone
  • using beat timing directly instead of hoping the model infers it
  • adding a short personal baseline only as calibration, not as a replacement for the generic model

What changed the story the most

  • moving away from optimistic random splits
  • using the de Chazal inter-patient split for unseen-patient testing
  • balancing validation folds by ventricular beat counts
  • tracking latency every time the model got stronger

Current Architecture

The best-performing personalized version so far keeps a strong generic detector and adds a small past-only calibration branch. 

ECG record
  |
  +--> R-peak aligned beat windows
          |
          +--> local beat neighborhood
          |      |
          |      +--> multiscale morphology encoder
          |      +--> short-range rhythm encoder
          |      +--> RR features
          |      +--> RR-normalized center beat
          |
          +--> causal history from the same record
                 |
                 +--> history morphology prototype
                 +--> rolling RR baseline
          |
          +--> fuse everything
          |
          +--> ventricular beat vs normal beat

generic model   = everything above except the causal history block
personalized    = generic model + past-only personal calibration
ablation target = RR baseline only vs prototype only vs both together

Main Techniques We Used

These are the ideas that really mattered after the testing setup got stricter.

Multiscale morphology The model looks at short, medium, and wider waveform patterns at the same time, which helps when heartbeats are stretched or compressed across patients.
Short-range rhythm context A beat is easier to judge when the model can see the beats right around it instead of treating it as a totally isolated waveform.
RR-aware features The model uses timing information directly, so it can notice when a beat arrives earlier or later than the recent rhythm would suggest.
RR-normalized beat view A second beat view is rescaled to the local heart cycle, which makes the model less tied to one fixed heart rate.
Beat-balanced validation folds This was one of the biggest fixes. Validation folds had to be balanced by ventricular beat counts, not just by which records were assigned where.
Causal personalization The personalized branch only looks backward in the same record. It gives the model a short-term idea of what is normal for that person right now.

Result Timeline

This is the short version of the path from easy-looking results to stricter and more believable ones. The last column explains the actual reason each stage moved, not just whether it was "better" or "worse."

Stage Scope Main result What it told us
Random split quick check Easy early setting baseline best F1 0.9922, attention best F1 0.9726 Train and test records were too friendly to each other, so the model could partly rely on patient-specific shortcuts instead of proving it could handle a truly new patient.
First strict context model de Chazal, fold 0, 3 seeds mean P 0.7739 / R 0.8693 / F1 0.8147 Once the test patients were really unseen, single-beat shape alone was not enough. The drop showed that this task depends on both morphology and local rhythm, not just memorizing familiar waveform styles.
Beat-balanced context de Chazal, 5 folds x 3 seeds mean P 0.7901 / R 0.9008 / F1 0.8348 Some validation folds used to contain many ventricular beats while others had very few, which made checkpoint choice noisy. Balancing folds by beat counts made validation feedback fairer, so the result became more stable for a real reason.
RR-aware generic model de Chazal, 5 folds x 3 seeds mean P 0.7995 / R 0.9500 / F1 0.8629, best single F1 0.9420 The model improved when it stopped looking only at shape and started looking directly at timing: whether a beat came early or late relative to nearby beats. That matches how ventricular beats are often recognized in practice.
Personalized model de Chazal, 5 folds x 3 seeds mean P 0.8177 / R 0.9676 / F1 0.8817, best single F1 0.9517 Giving the model a short past-only picture of what is normal for this record reduces confusion caused by patient-to-patient differences. It helps because the model is no longer judging every beat against a one-size-fits-all normal, but that extra personal calibration also costs extra compute.

What We Can Reasonably Say Right Now

What looks solid

Random split was too optimistic. The de Chazal inter-patient setup is the right main benchmark, and the generic RR-aware model is currently the best-performing generic direction so far under the stricter setting.

What still looks promising

Personal calibration helps on average and clearly helps in some runs, but it should be reported as a separate setting because it uses patient history.

What still needs work

Latency is still the main cost of personalization, and some folds still show that precision is not equally easy everywhere.
